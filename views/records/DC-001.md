## Context

PKMP の主要概念 (UDR, Document/Record 分離, Registry, Capability 交換可能性等) は AXIS の既存概念と強い重なりがある。PKMP は SRM の前提として発生し、独立した目的を持つ。同じ概念 (特に UDR) が両プロジェクトに並存する状態。

## Decision

short_term: PKMP を AXIS から独立したプロジェクトとして設計・実装する
long_term: PKMP が成熟した段階で AXIS との統合可能性を評価する

## Rationale

- PKMP は Knowledge Modeling の Meta Layer 候補であり、AXIS に従属させると Meta Layer として機能できなくなる
- AXIS の都合で PKMP 設計が歪むことを防ぐ
- SRM を独立 Reference Project として PKMP を検証できる
- 統合は概念整理が固まってから判断すべき

## Implications

- UDR 概念は当面 PKMP と AXIS で別実装として並存することを許容する
- PKMP の Repository 構造設計時、AXIS の用語との衝突は気にしない
- 将来統合時の Migration Path は後日 Decision Record として記録する

## Rejected Alternatives

### PKMP を AXIS のサブプロジェクトにする (alt-a)

Meta Layer になれない; AXIS の都合で歪む

### PKMP と AXIS を最初から統合設計する (alt-b)

両者の成熟度が違う; AXIS 側に統合準備がない
