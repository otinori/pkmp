## Context

PKMP's self-improvement loop triggers when an SDD change is archived (e.g., via
openspec-archive-change). Simulation Round 14 (Sim 75) confirmed that the
openspec-archive-change skill has no reference to PKMP or capture-learnings. The
connection between "change archived" and "run capture-learnings" exists only in
CLAUDE.md's and AGENTS.md's trigger tables.


## Learning

PKMP's integration with SDD tools is event-driven but not event-dispatched.
When an SDD change is archived:
1. The SDD tool (openspec) completes its archiving action and reports success
2. The AI agent MUST recognize the archive event as a PKMP trigger
3. The AI agent MUST then invoke /pkmp:capture-learnings as the next step
4. If the AI agent doesn't read the trigger table, step 3 is silently skipped

This means the loop closure quality depends entirely on:
- The AI agent seeing and understanding CLAUDE.md/AGENTS.md trigger tables
- The AI agent proactively chaining tools after SDD events

There is no automatic hook, no event bus, no callback mechanism.


## Applicability

When evaluating PKMP adoption: ensure the AI agent configuration reads CLAUDE.md
or AGENTS.md at session start. For custom AI setups (non-Claude Code), verify
that the trigger table is accessible and the AI agent actively checks it after
completing SDD operations.

When extending PKMP to new SDD tools: document the trigger in the PKMP trigger
table rather than modifying the SDD tool's skill — keep tool boundaries clean.
